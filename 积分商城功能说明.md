# 积分商城功能说明文档

## 功能概述

积分商城是一个完整的积分兑换系统，用户可以使用积分兑换各种商品和服务。系统包括前台用户界面和后台管理界面。

## 功能模块

### 1. 前台用户功能

#### 1.1 积分商城页面 (`/Front/IntegralMall`)
- **位置**: 前台导航菜单 -> 积分商城
- **功能**:
  - 展示用户当前积分余额
  - 浏览所有可兑换的商品
  - 按商品名称和分类搜索商品
  - 点击"兑换"按钮进行积分兑换
  - 实时显示兑换后的积分余额

#### 1.2 商品展示
- 商品卡片展示，包含:
  - 商品图片
  - 商品名称
  - 商品描述
  - 所需积分
  - 兑换按钮（积分不足时禁用）

#### 1.3 兑换流程
1. 用户点击商品的"兑换"按钮
2. 弹出确认对话框，显示:
   - 商品名称
   - 所需积分
   - 当前积分
   - 兑换后积分
3. 用户确认后完成兑换
4. 系统自动扣除积分并记录兑换历史

### 2. 后台管理功能

#### 2.1 积分商城管理页面 (`/Admin/IntegralMallAdmin`)
- **位置**: 后台导航菜单 -> 积分商城
- **权限**: 仅管理员可访问

#### 2.2 商品管理功能
- **查看商品列表**
  - 表格展示所有商品信息
  - 支持分页显示
  - 显示商品状态（上架/下架）

- **搜索和筛选**
  - 按商品名称搜索
  - 按商品分类筛选
  - 支持重置搜索条件

- **新增商品**
  - 点击"新增商品"按钮
  - 填写商品信息:
    - 商品名称（必填）
    - 商品分类（必填）：优惠券/礼品/服务
    - 积分价格（必填）
    - 库存数量
    - 商品描述
    - 商品图片URL
    - 上架/下架状态
  - 点击保存完成新增

- **编辑商品**
  - 点击表格中的"编辑"按钮
  - 修改商品信息
  - 点击保存完成编辑

- **删除商品**
  - 点击表格中的"删除"按钮
  - 确认删除后完成操作

## 数据库表结构

### 1. 商品表 (integral_mall_product)
```sql
- id: 商品ID (主键)
- product_name: 商品名称
- category: 商品分类 (coupon/gift/service)
- integral_price: 积分价格
- stock: 库存数量
- description: 商品描述
- image_url: 商品图片URL
- status: 状态 (1:上架, 0:下架)
- create_time: 创建时间
- update_time: 更新时间
```

### 2. 兑换记录表 (integral_mall_exchange_record)
```sql
- id: 记录ID (主键)
- user_id: 用户ID
- product_id: 商品ID
- product_name: 商品名称
- integral_price: 消耗积分
- quantity: 兑换数量
- exchange_time: 兑换时间
- status: 状态 (1:成功, 0:失败)
- remark: 备注
```

## 后端API接口

### 1. 前台接口

#### 获取商品列表
```
POST /IntegralMall/GetProductList
参数:
  - productName: 商品名称 (可选)
  - category: 商品分类 (可选)
返回: 商品列表
```

#### 购买商品
```
POST /IntegralMall/BuyProduct
参数:
  - productId: 商品ID (必填)
  - quantity: 购买数量 (必填)
返回: 成功/失败信息
```

### 2. 后台接口

#### 获取商品列表（分页）
```
POST /IntegralMall/AdminGetProductList
参数:
  - productName: 商品名称 (可选)
  - category: 商品分类 (可选)
  - pageNum: 页码 (必填)
  - pageSize: 每页数量 (必填)
返回: 商品列表和总数
```

#### 新增商品
```
POST /IntegralMall/AddProduct
参数:
  - productName: 商品名称 (必填)
  - category: 商品分类 (必填)
  - integralPrice: 积分价格 (必填)
  - stock: 库存 (必填)
  - description: 描述 (可选)
  - imageUrl: 图片URL (可选)
  - status: 状态 (必填)
返回: 成功/失败信息
```

#### 编辑商品
```
POST /IntegralMall/UpdateProduct
参数: 同新增商品，需要包含id
返回: 成功/失败信息
```

#### 删除商品
```
POST /IntegralMall/DeleteProduct
参数:
  - id: 商品ID (必填)
返回: 成功/失败信息
```

## 业务流程

### 用户兑换流程
```
用户登录 -> 进入积分商城 -> 浏览商品 -> 选择商品 -> 确认兑换 
-> 系统扣除积分 -> 记录兑换历史 -> 兑换成功
```

### 管理员管理流程
```
管理员登录 -> 进入积分商城管理 -> 查看/搜索商品 
-> 新增/编辑/删除商品 -> 管理完成
```

## 注意事项

1. **积分扣除**
   - 兑换时需要检查用户积分是否充足
   - 扣除积分后需要记录到积分记录表
   - 兑换失败时不应扣除积分

2. **库存管理**
   - 每次兑换时需要检查商品库存
   - 库存不足时应提示用户
   - 兑换成功后需要更新库存

3. **数据安全**
   - 所有操作需要进行权限验证
   - 用户只能查看自己的兑换记录
   - 管理员操作需要记录日志

4. **性能优化**
   - 商品列表建议添加缓存
   - 频繁查询的数据可以使用索引优化
   - 大量兑换时考虑使用消息队列

## 常见问题

**Q: 用户积分不足时如何处理？**
A: 系统会在兑换按钮上显示"积分不足"，并禁用按钮，防止用户进行兑换操作。

**Q: 商品库存为0时如何处理？**
A: 建议在后台管理中将商品下架，或在前台显示"库存不足"提示。

**Q: 兑换失败后积分会被扣除吗？**
A: 不会。系统应该在扣除积分前进行所有验证，确保兑换成功后再扣除积分。

**Q: 如何查看用户的兑换历史？**
A: 可以在后台添加"兑换记录"管理页面，查询 `integral_mall_exchange_record` 表。
