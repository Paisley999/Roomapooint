## 📦 后端代码集成步骤

### 1. 创建Entity类

**文件位置**: `src/main/java/com/yourpackage/entity/`

创建两个实体类：
- `IntegralProduct.java` - 商品实体
- `IntegralExchange.java` - 兑换记录实体

参考 `积分商城后端实现(修改版)_Part1.java` 中的Entity部分

### 2. 创建Mapper接口

**文件位置**: `src/main/java/com/yourpackage/mapper/`

创建两个Mapper接口：
- `IntegralProductMapper.java`
- `IntegralExchangeMapper.java`

参考 `积分商城后端实现(修改版)_Part1.java` 中的Mapper部分

### 3. 创建Service类

**文件位置**: `src/main/java/com/yourpackage/service/`

创建 `IntegralMallService.java` 服务类

参考 `积分商城后端实现(修改版)_Part1.java` 中的Service部分

**关键依赖注入**：
```java
@Autowired
private IntegralProductMapper productMapper;

@Autowired
private IntegralExchangeMapper exchangeMapper;

@Autowired
private IntegralService integralService;  // 你现有的积分服务
```

### 4. 创建Controller类

**文件位置**: `src/main/java/com/yourpackage/controller/`

创建 `IntegralMallController.java` 控制器

参考 `积分商城后端实现(修改版)_Part2.java` 中的Controller部分

**重要**: 修改 `getCurrentUserId()` 方法以适配你的项目：

```java
private Integer getCurrentUserId() {
    // 根据你的项目实际情况获取当前用户ID
    // 示例1: 从Session获取
    HttpServletRequest request = ((ServletRequestAttributes) 
        RequestContextHolder.getRequestAttributes()).getRequest();
    return (Integer) request.getSession().getAttribute("userId");
    
    // 示例2: 从Token获取
    // String token = request.getHeader("Authorization");
    // return parseUserIdFromToken(token);

```

### 5. 创建异常和返回类

**文件位置**: `src/main/java/com/yourpackage/exception/` 和 `src/main/java/com/yourpackage/dto/`

- `BusinessException.java` - 业务异常

## 🔗 与现有系统的集成

### 1. 积分系统集成

确保你的 `IntegralService` 有以下方法：

```java
// 获取用户积分
```

### 2. 用户系统集成

确保 `AppUser` 实体和 `AppUserMapper` 已存在

### 3. MyBatis-Plus配置

确保你的项目已配置MyBatis-Plus：

```xml

<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-boot-starter</artifactId>
    <version>3.4.1</version>
</dependency>
```

## 📝 API接口列表

### 前台接口

| 接口 | 方法 | 说明 | 参数 |
|------|------|------|------|
| `/IntegralMall/GetProductList` | POST | 获取商品列表 | productName(可选) |
| `/IntegralMall/BuyProduct` | POST | 购买商品 | userId, productId, quantity |
| `/IntegralMall/GetUserExchangeRecords` | POST | 获取用户兑换记录 | userId |

### 后台接口

| 接口 | 方法 | 说明 | 参数 |
|------|------|------|------|
| `/IntegralMall/AdminGetProductList` | POST | 获取商品列表(分页) | productName, pageNum, pageSize |
| `/IntegralMall/AddProduct` | POST | 新增商品 | productName, productDescription, requiredIntegral, stock, imageUrl, status, sortOrder |
| `/IntegralMall/UpdateProduct` | POST | 编辑商品 | id, productName, productDescription, requiredIntegral, stock, imageUrl, status, sortOrder |
| `/IntegralMall/DeleteProduct` | POST | 删除商品 | id |
| `/IntegralMall/GetPendingShipments` | POST | 获取待发货订单 | 无 |
| `/IntegralMall/ShipOrder` | POST | 发货 | exchangeId, trackingNumber, remark |
| `/IntegralMall/ConfirmReceipt` | POST | 确认收货 | exchangeId |

## ⚠️ 注意事项

### 1. 事务管理

所有修改操作都使用了 `@Transactional` 注解，确保数据一致性

### 2. 积分扣除

购买商品时会调用 `IntegralService.deductIntegral()` 方法，确保该方法已正确实现

### 3. 库存管理

- 购买前检查库存
- 购买成功后自动扣除库存
- 库存不足时拒绝购买

### 4. 状态管理

- 商品状态：1=上架, 2=下架
- 兑换状态：1=待发货, 2=已发货, 3=已收货

### 5. 权限控制

- 后台接口需要管理员权限
- 前台接口需要用户登录

## 📞 常见问题

### Q1: 如何获取当前用户ID？

A: 根据你的项目认证方式修改 `getCurrentUserId()` 方法

### Q2: 如何处理积分不足的情况？

A: 系统会抛出 `BusinessException("积分不足")`，前端会收到错误信息

### Q3: 如何查看用户的兑换记录？

A: 调用 `/IntegralMall/GetUserExchangeRecords` 接口

### Q4: 如何管理商品库存？

A: 在后台通过 `/IntegralMall/UpdateProduct` 接口修改库存
