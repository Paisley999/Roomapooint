# 积分商城 - 快速参考卡片

## 📊 数据库表结构速查

### integral_product（商品表）
```
Id (PK)              - 商品ID
ProductName          - 商品名称 ✓
ProductDescription   - 商品描述
RequiredIntegral     - 所需积分 ✓
Stock                - 库存数量 ✓
ImageUrl             - 商品图片
Status               - 状态(1=上架, 2=下架) ✓
SortOrder            - 排序
CreationTime         - 创建时间
CreatorId (FK)       - 创建人
```

### integral_exchange（兑换记录表）
```
Id (PK)              - 记录ID
UserId (FK)          - 用户ID ✓
ProductId (FK)       - 商品ID ✓
ExchangeIntegral     - 兑换所用积分 ✓
ExchangeStatus       - 状态(1=待发货, 2=已发货, 3=已收货)
CreationTime         - 兑换时间
ShippingTime         - 发货时间
ShippedBy (FK)       - 发货人
ReceivedTime         - 收货时间
TrackingNumber       - 物流单号
ShippingRemark       - 发货备注
```

## 🌐 API端点速查

```

  └─ 返回: List<IntegralProduct>
```

## 🔄 业务流程

### 用户兑换流程
```
1. 用户浏览商品 → GetProductList
2. 用户选择商品 → 显示商品详情
3. 用户点击兑换 → BuyProduct
4. 系统验证：
   ├─ 商品是否存在
   ├─ 商品是否上架
   ├─ 库存是否充足
   └─ 用户积分是否充足
5. 系统执行：
   ├─ 扣除用户积分
   ├─ 更新商品库存
   └─ 创建兑换记录
6. 返回成功信息
```

### 管理员发货流程
```
1. 查看待发货订单 → GetPendingShipments
2. 选择订单 → 填写物流信息
3. 点击发货 → ShipOrder
4. 系统更新订单状态为"已发货"
5. 用户收货 → ConfirmReceipt
6. 系统更新订单状态为"已收货"
```

## 🚨 错误处理

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| 商品不存在 | 商品ID无效 | 检查商品ID |
| 商品已下架 | 商品状态为2 | 联系管理员 |
| 商品库存不足 | 库存数量不够 | 等待补货 |
| 积分不足 | 用户积分不够 | 获取更多积分 |
| 订单不存在 | 订单ID无效 | 检查订单ID |
| 订单状态不正确 | 订单状态不允许操作 | 检查订单状态 |

## 📞 快速问题排查

**Q: 兑换失败，提示"积分不足"**
A: 检查用户积分是否足够，调用IntegralService.getUserIntegral()验证

**Q: 商品显示为空**
A: 检查商品状态是否为1（上架），检查库存是否大于0

**Q: 发货失败**
A: 检查订单状态是否为1（待发货），检查exchangeId是否正确

**Q: 无法获取当前用户ID**
A: 修改getCurrentUserId()方法，根据项目认证方式获取用户ID
